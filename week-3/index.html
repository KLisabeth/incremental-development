ARIA


Accessible Rich Internet Applications

A quick video intro

What is ARIA?


  1. It is specialized attributes in HTML
  2. Like CSS but for the accessibility tree
  3. Adds a layer of description over your HTML
  4. It modifies how readers interpret the DOM
  5. No ARIA is better than bad ARIA!

Accessibility Inspector


  1. Open Firefox Devtools
  2. Turn on the "Accessibility" tab
  3. Thats the Accessibility Tree!
  4. It's how Screen Readers understand a website

Practical Tips


Ideas for improving your accessible development

Plan for accessibility


Good design is accessible design (short video)

Start at the source


  • Use The Web Accessibility plugin for VSC
  • Write semantic HTML
  • Always use the right tag
  • Keep your HTML as simple as possible

Inspect & Audit


  • Turn on firefox's accessibility tree inspector
  • Test your site with Chrome/ium's Lighthouse audit
  • Use extensions like Wave or tota11y

Test it yourself!


  • Use your site with noCoffee or a screen reader.
  • Emulate a tiny phone with slow internet.
  • Disable CSS
  • Try navigating with only your keyboard.
  • Use your computer's Screen Reader

Code-Along


Build a simple project together to see the new material in action.


(~80 min)

for this second part of class, take the time to cover whatever the students need most. It's not a rush to finish the project

Take plenty of time to cover the committing, branching & push/pulling. If you need to pause and do extra examples on Git or anything else that is great

if you're feeling that the project is moving too fast, it can help to pause from coding together. Have them work together in small groups, walk around, answer their questions, then get back to group coding.

students find it very helpful if you record this part of class

Objectives:


  • Start from an empty project
  • Reverse-engineer a design from an existing site
  • Creating a wireframe
  • Plan steps, developing each on a branch
  • Use semantic HTML5 elements & ARIA
  • Use Accessibility tools to inspect and audit
  • Pulling, Pushing & GitHub Pages

Set-up


  1. clone this repo (if you haven't yet)
  2. Open the project folder in VSC
  3. Open development-strategy.md for planning
  4. Open Google for reverse-engineering
  5. Open index.html & style.css in VSC
  6. Open them in Firefox & enable the Accessibility tools
  7. Begin!
It is important everyone uses firefox or they won't be able to inspect the accessibility tree.
